## 了解 CLI 和 IDE 的协作模式

### IDE 界面（CodeBuddy）
- 配置子 Agent（`.codebuddy/agents/`）、Skills、Rules、Commands
- 适合需要讨论、看 diff、精细调整的交互式主任务
- 单会话串行，生成回复时无法插入新消息

### CLI 界面（cbc / claude）
- 在终端中运行，多终端 Tab 即可实现多 Agent 并行
- 适合目标明确、可以"丢出去跑"的独立任务（迁移生成、批量重构、code review 等）
- CLI 会读取项目目录下的 `.codebuddy/` 配置，与 IDE 共享 Agent/Skill/Rule

### 推荐工作流：IDE + CLI 混合使用
- **IDE**：交互式推进主 feature（如 `/speckit.implement`）
- **CLI**：并行处理不冲突的任务（如另一个 feature 或 code review）
- **核心原则**：确保并行任务修改的文件不重叠，避免冲突

### 终端选择
| 终端 | 优点 | 适用场景 |
|------|------|---------|
| CodeBuddy 内置终端 | 同窗口切换方便，共享工作目录 | `npm run dev`、`git` 等常规命令 |
| iTerm2 | 分屏强大，稳定可靠 | CLI Agent 并行任务 |
| Warp | AI 辅助补全，输出分块，体验现代 | CLI Agent 长输出场景 |

建议日常命令用内置终端，CLI Agent 并行任务放外部终端（iTerm2 / Warp），避免挤占 IDE 屏幕空间。
